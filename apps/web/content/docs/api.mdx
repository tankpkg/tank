# API Reference

Base URL (hosted):

```text
https://tankpkg.dev/api/v1
```

Authentication for protected endpoints uses Bearer API keys.

## CLI Auth Endpoints

| Endpoint | Method | Purpose |
|---|---|---|
| `/cli-auth/start` | POST | Start browser-auth flow |
| `/cli-auth/authorize` | GET | User authorization step |
| `/cli-auth/exchange` | POST | Exchange poll token for API key |

## Registry Endpoints

| Endpoint | Method | Purpose |
|---|---|---|
| `/skills` | GET | Search/list skills |
| `/skills` | POST | Start publish flow |
| `/skills/confirm` | POST | Confirm finalize publish |
| `/skills/[name]` | GET | Skill metadata |
| `/skills/[name]/versions` | GET | List versions |
| `/skills/[name]/star` | POST/DELETE | Star/unstar |
| `/skills/[name]/[version]` | GET | Version metadata |
| `/skills/[name]/[version]/files/[...path]` | GET | File content retrieval |
| `/search` | GET | Search endpoint |

## Minimal Usage Example

```bash
curl -H "Authorization: Bearer $TANK_API_KEY"   https://tankpkg.dev/api/v1/skills/my-org/my-skill
```

## Stability Guidance

- Treat endpoint paths above as canonical.
- For payload details, validate against current route handlers in `apps/web/app/api/v1/**`.
- Keep clients resilient to additive response fields.
