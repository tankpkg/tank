# CLI Reference

Source of truth: `apps/cli/src/bin/tank.ts`.

## Commands

| Command | Args | Options | Purpose |
|---|---|---|---|
| `tank init` | - | - | Initialize `skills.json` |
| `tank login` | - | - | Authenticate via browser |
| `tank whoami` | - | - | Show current user |
| `tank logout` | - | - | Clear auth token |
| `tank publish` | - | `--dry-run`, `--private`, `--visibility <mode>` | Publish a skill |
| `tank install` | `[name] [version-range]` | `-g, --global` | Install one/all skills |
| `tank remove` | `<name>` | `-g, --global` | Remove installed skill |
| `tank update` | `[name]` | `-g, --global` | Update one/all skills |
| `tank verify` | - | - | Verify installation integrity |
| `tank permissions` | - | - | Show resolved permissions |
| `tank search` | `<query>` | - | Search registry |
| `tank info` | `<name>` | - | Show package details |
| `tank audit` | `[name]` | - | Run/read security audit |
| `tank link` | - | - | Link local skills for agents |
| `tank unlink` | - | - | Remove links |
| `tank doctor` | - | - | Diagnose environment/setup |

## Typical Workflows

### Consumer

```bash
tank login
tank install @org/skill-name
tank permissions
tank verify
```

### Publisher

```bash
tank login
tank init
tank publish --dry-run
tank publish
```

### Maintenance

```bash
tank update
tank audit
tank doctor
```

## Notes

- Use `tank <command> --help` for runtime help text.
- Commands not listed above are not part of current CLI entrypoint.
