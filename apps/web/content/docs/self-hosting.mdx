# Self-Hosting

This runbook is aligned to the current monorepo layout.

## Architecture

- **Web app**: `apps/web` (Next.js)
- **Scanner**: `python-api` (FastAPI pipeline)
- **Shared contracts**: `packages/shared`
- **Storage**: Supabase or compatible object storage
- **Database**: PostgreSQL

## Prerequisites

- Node.js 24+
- pnpm 10+
- Python 3.14+
- PostgreSQL instance
- Storage credentials

## Minimum Setup

```bash
pnpm install
cp .env.example .env.local
```

Required envs (minimum):

- `DATABASE_URL`
- `BETTER_AUTH_SECRET`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `PYTHON_API_URL`

## Build and Run

```bash
pnpm turbo build --filter=@tank/web...
pnpm --filter=@tank/web start
```

## Schema / Migrations

```bash
pnpm --filter=web drizzle-kit generate
pnpm --filter=web drizzle-kit push
```

## Admin Bootstrap

```bash
pnpm --filter=web admin:bootstrap
```

## Health and Verification

- Web health endpoint: `/api/health`
- Validate publish/install flows with CLI in a clean environment
- Confirm scanner connectivity via `PYTHON_API_URL`

## Operational Notes

- Keep `python-api/` and `apps/web/api-python/` in sync when scanner logic changes.
- Use Turbo builds for dependency-aware monorepo builds.
- Do not bypass auth secret requirements in production.

## Troubleshooting

### Build fails with workspace package resolution

Use monorepo-aware build command:

```bash
pnpm turbo build --filter=@tank/web...
```

### Auth runtime errors

Set `BETTER_AUTH_SECRET` explicitly in deployment environment.

### Scanner errors

Verify `PYTHON_API_URL` and scanner deployment health.
