# Self-Host in 15 Minutes

Deploy Tank on your own infrastructure for complete control over your AI skill registry and security scanning.

## Why Self-Host?

- **Data sovereignty** вҖ” Skills and metadata never leave your infrastructure
- **Air-gapped support** вҖ” Deploy in environments without internet access
- **Custom policies** вҖ” Implement organization-specific security rules
- **Compliance** вҖ” Meet regulatory requirements (SOC2, HIPAA, etc.)

## Prerequisites

- Docker and Docker Compose
- 4GB RAM minimum (8GB recommended)
- PostgreSQL 17+ (or use provided Docker config)
- S3-compatible storage (or MinIO)

## Step 1: Clone and Configure

```bash
git clone https://github.com/tankpkg/tank.git
cd tank
cp .env.example .env.local
```

Edit `.env.local` with your values:

```bash
# Database
DATABASE_URL=postgresql://tank:password@localhost:5432/tank

# Storage (choose one)
STORAGE_PROVIDER=onprem  # or 'supabase'
ONPREM_STORAGE_PATH=/data/tank-storage

# Auth
NEXT_PUBLIC_APP_URL=https://tank.yourcompany.com
GITHUB_CLIENT_ID=your-github-oauth-id
GITHUB_CLIENT_SECRET=your-github-oauth-secret

# Security
FIRST_ADMIN_EMAIL=admin@yourcompany.com
```

## Step 2: Start Services

Using Docker Compose:

```bash
docker-compose up -d
```

This starts:
- Next.js web application (port 3000)
- PostgreSQL database (port 5432)
- MinIO S3-compatible storage (port 9000)
- Python security scanner (internal)

## Step 3: Run Database Migrations

```bash
docker-compose exec web pnpm drizzle-kit push
```

## Step 4: Create Admin User

```bash
docker-compose exec web pnpm admin:bootstrap
```

Promotes `FIRST_ADMIN_EMAIL` to admin role.

## Step 5: Configure OIDC SSO (Optional)

For enterprise single sign-on, add to `.env.local`:

```bash
OIDC_ISSUER=https://sso.yourcompany.com
OIDC_CLIENT_ID=tank-client
OIDC_CLIENT_SECRET=your-oidc-secret
```

## Step 6: Verify Installation

```bash
# Check all services are running
docker-compose ps

# Test the API
curl http://localhost:3000/api/v1/skills

# Test security scanner
curl -X POST http://localhost:3000/api/v1/scan \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com/skill.tar.gz"}'
```

## High Availability Setup

For production deployments:

### Database (PostgreSQL)

```yaml
# Use managed PostgreSQL (RDS, Cloud SQL, etc.)
DATABASE_URL=postgresql://tank:password@postgres.example.com:5432/tank?sslmode=require
```

### Storage (S3)

```yaml
# Use AWS S3 or compatible
STORAGE_PROVIDER=onprem
AWS_ACCESS_KEY_ID=your-access-key
AWS_SECRET_ACCESS_KEY=your-secret-key
AWS_REGION=us-east-1
S3_BUCKET=tank-skills
```

### Container Orchestration

Deploy to Kubernetes using provided Helm charts:

```bash
helm install tank ./infra/helm/tank \
  --set database.url=$DATABASE_URL \
  --set storage.bucket=$S3_BUCKET
```

## Monitoring

Access included observability stack:

- **Grafana**: `http://localhost:3001` (admin/admin)
- **Loki**: Log aggregation
- **Prometheus**: Metrics collection

Default dashboards:
- Tank API performance
- Security scan latency
- Skill publish/download rates

## Upgrading

```bash
git pull origin main
docker-compose pull
docker-compose up -d
```

Migrations run automatically on startup.
