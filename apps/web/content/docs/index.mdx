# Tank Documentation

Tank is a security-first package manager for AI agent skills.

Use this docs set by intent:

- **Learn fast**: [Getting Started](/docs/getting-started)
- **Ship a skill**: [Publishing](/docs/publishing)
- **Use skills in agents**: [Installing](/docs/installing)
- **AI Editor Integration**: [MCP Server](/docs/mcp)
- **Command lookup**: [CLI Reference](/docs/cli)
- **Integration lookup**: [API Reference](/docs/api)
- **Run your own registry**: [Self-Hosting](/docs/self-hosting)

## Product Guarantees

- SHA-512 integrity verification on install
- Mandatory security scanning before publish
- Permission declarations in `skills.json`
- Deterministic lockfile workflow

## Choose Your Path

### I am a skill publisher

1. Follow [Getting Started](/docs/getting-started)
2. Implement your manifest and code via [Publishing](/docs/publishing)
3. Validate with `tank doctor` and `tank publish --dry-run`
4. Publish and verify via `tank info @org/name`

### I am a skill consumer

1. Install a package with [Installing](/docs/installing)
2. Review permissions with `tank permissions`
3. Verify integrity with `tank verify`

### I want to use Tank from my AI editor

1. Set up the [MCP Server](/docs/mcp) in Claude Code, Cursor, or VS Code
2. Ask your AI to scan, publish, or search skills directly

### I need implementation details

- Commands: [CLI Reference](/docs/cli)
- Endpoints: [API Reference](/docs/api)
- Ops: [Self-Hosting](/docs/self-hosting)
