# Installing Skills

This guide covers day-2 operations for skill consumers.

## Install

```bash
tank install @org/skill-name
tank install @org/skill-name '^1.2.0'
```

Optional global mode:

```bash
tank install @org/skill-name '*' -g
```

## Update

```bash
tank update
tank update @org/skill-name
```

Optional global mode:

```bash
tank update @org/skill-name -g
```

## Remove

```bash
tank remove @org/skill-name
tank remove @org/skill-name -g
```

## Verify and Audit

```bash
tank verify
tank permissions
tank audit
tank audit @org/skill-name
```

## Discover and Inspect

```bash
tank search "embedding"
tank info @org/skill-name
```

## Operational Guidance

- Run `tank verify` after changes to lock/install state
- Run `tank permissions` before enabling skills in production agents
- Prefer pinned versions for production stability

## Failure Handling

### Install fails

Run `tank doctor`; check auth and network.

### Unexpected permission scope

Remove package and select an alternative with narrower access.

### Verify fails

Reinstall package and ensure lock state is consistent.
