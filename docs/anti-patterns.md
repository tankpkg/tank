# Anti-Patterns

Rules that prevent wasted time, broken builds, and security incidents. The top 5 are in AGENTS.md. This is the full list.

## Universal

- **Never suppress types** — no `as any`, `@ts-ignore`, `@ts-expect-error`. Fix the type, don't hide it.
- **Never use `.spec.ts`** — always `.test.ts`. Consistency across all packages.
- **Never import between packages** — CLI, Web, MCP server must not import from each other. Use `@internal/shared`. Cross-package imports break the dependency graph and create circular deps.
- **Never commit `.env.local`** — contains real credentials. Copy from `.env.example`.
- **Never use `require()`** — ESM only across all packages.
- **Never refactor while bugfixing** — fix minimally, ship it, refactor in a separate PR. If a fix touches >3 files, question scope.
- **Never use Zod `parse()`** — always `safeParse()`. `parse()` throws, and thrown exceptions are invisible control flow that bypasses error handling.

## Web App

- **Never create DB connections outside `lib/db.ts`** — `globalThis` singleton prevents connection leaks during Next.js hot-reload.
- **Never use Supabase for DB queries** — Supabase is file storage only (tarballs). All queries go through Drizzle ORM to PostgreSQL.
- **Never expose `supabaseAdmin` to browser** — service-role key is server-only. Leaking it = full storage access.
- **Never put auth checks in page components** — use layout-level guards in route groups.
- **Never use raw SQL or Prisma** — Drizzle ORM only. Raw SQL is error-prone, Prisma has too much magic.
- **Never modify `auth-schema.ts` manually** — auto-generated by better-auth. Edits get overwritten.
- **Never skip audit logging** for admin actions — compliance and incident investigation depend on it.

## CLI & MCP Server

- **Never hardcode registry URL** — use `REGISTRY_URL` from `@internal/shared` or user config. Hardcoding breaks on-prem deployments.
- **Never skip SHA-512 verification** during install — integrity check prevents tampered tarballs.
- **Never extract without security filters** — reject symlinks, hardlinks, path traversal (`../`), absolute paths. These are common supply chain attack vectors.
- **Never exceed tarball limits** — 1000 files or 50MB. Enforced in packer. Prevents zip bombs and resource exhaustion.

## Python Scanner

- **Never skip stage0 (ingest)** — all subsequent stages depend on its output (file hashes, extracted content).
- **Never swallow stage errors silently** — use `errored` status, not empty results. Silent failures hide security gaps.

## Shared Package

- **Never add side-effect dependencies** — this package must stay pure. Side effects break tree-shaking and make testing unpredictable.
- **Never mutate exported constants** — treat as frozen. Consumers depend on immutability.
