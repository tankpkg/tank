# IDD

Intent-driven development lives here.

## Structure

- `idd/modules/<capability>/INTENT.md` — capability-level intent, constraints, examples
- `idd/active/<initiative>.md` — active cross-cutting initiatives and migration plans
- `idd/templates/` — reusable intent templates

Use IDD before BDD/TDD/E2E for new capabilities, migrations, auth changes, and behavior changes with user impact.
